# モジュールガイド

各 SPM モジュール（ターゲット）の責務・公開 API・依存関係を解説し、新メンバーがモジュール構成を理解できるようにする。

## ツール使用方針

- `Package.swift` の読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read ツールで `Package.swift` を直接読み取る
- モジュール内のファイル探索には Glob / Grep を使用する

## 分析手順

1. `Package.swift` を読み取り、全ターゲットとその依存関係を抽出する
2. 各ターゲットのソースディレクトリを特定する
3. 各モジュール内の `public` な型・関数を Grep で収集し、公開 API を把握する
4. 各モジュール内の `import` 文を収集し、実際の依存関係を確認する
5. モジュールの責務をファイル内容から推定する
6. モジュール間の依存方向を整理する

## 出力形式

```
## モジュールガイド

### モジュール依存図
<テキストベースの依存関係図>

### 各モジュール詳細

#### <モジュール名>
- **責務**: <モジュールの主な責務>
- **種別**: <Library / Executable / Plugin>
- **依存先**: <依存するモジュール一覧>
- **依存元**: <このモジュールに依存するモジュール一覧>
- **公開 API**:
  - `<型名>` - <概要>
  - `<関数名>` - <概要>
- **主要ファイル**:
  - `<ファイルパス>` - <役割>

### モジュール間の設計方針
- <依存方向のルール>
- <共通モジュールの扱い>
- <Feature モジュールの粒度>
```

## 対象の特定

- 引数でモジュール名が指定された場合はそのモジュールのみ詳細に解説する
- 指定がない場合は全モジュールの概要を生成する
