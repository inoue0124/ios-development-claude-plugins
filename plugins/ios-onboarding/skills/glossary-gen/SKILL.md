# 用語集生成

プロジェクトのソースコードから、ドメイン固有の用語・略語・独自の命名規則を抽出し、新メンバーが理解すべき用語集を自動生成する。

## ツール使用方針

- ファイル読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read / Glob / Grep ツールで直接ファイルを読み取る

## 抽出手順

1. Grep でプロジェクト内の主要な型定義を収集する
   - `struct`, `class`, `enum`, `protocol`, `actor` の名前
   - `typealias` の定義
2. 型名をキャメルケースで分解し、頻出する単語を抽出する
3. ドメイン固有の単語（一般的な Swift / iOS の語彙に含まれないもの）を特定する
4. 略語（大文字 2 文字以上の連続: `DTO`, `API`, `URL` 等）を抽出する
5. コメント内の `// MARK:` や ドキュメンテーションコメント（`///`）から説明文を収集する
6. `README.md` や `.md` ファイルからプロジェクト説明を補完する
7. 型の使用コンテキストから意味を推定する

## 用語の分類

| カテゴリ | 説明 | 例 |
|---|---|---|
| ドメインモデル | ビジネスドメインの主要概念 | `Order`, `Prescription`, `Invoice` |
| 略語 | プロジェクト内で使われる略語 | `Rx` (処方箋), `SKU` (在庫管理単位) |
| 技術用語 | プロジェクト固有の技術用語 | `DataStore` (独自のキャッシュ層) |
| 命名規則 | チーム固有の命名パターン | `*UseCase`, `*Repository`, `*DTO` |

## 出力形式

```
## プロジェクト用語集

### ドメインモデル
| 用語 | 推定される意味 | 定義箇所 |
|---|---|---|
| <用語> | <コンテキストから推定した意味> | <ファイルパス> |

### 略語一覧
| 略語 | 推定される正式名称 | 使用箇所 |
|---|---|---|
| <略語> | <推定> | <ファイルパス> |

### 技術用語
| 用語 | 説明 | 使用箇所 |
|---|---|---|
| <用語> | <プロジェクト内での使われ方> | <ファイルパス> |

### 命名規則パターン
| パターン | 意味 | 例 |
|---|---|---|
| `*ViewModel` | View の状態管理・UI ロジック | `UserViewModel`, `OrderViewModel` |
| `*Repository` | データアクセスの抽象化 | `UserRepository`, `OrderRepository` |
| `*UseCase` | ビジネスロジックの単位 | `FetchUserUseCase` |
| <その他検出されたパターン> | <推定> | <例> |
```

## 対象の特定

- 引数でモジュールやディレクトリが指定された場合はそのスコープ内の用語を収集する
- 指定がない場合はプロジェクト全体を対象にする
