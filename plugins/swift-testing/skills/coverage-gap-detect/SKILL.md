# テスト未カバーパスの特定

ソースコードとテストコードを突き合わせ、テストされていないコードパスを特定する。

## ツール使用方針

- ファイル読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read / Grep ツールで直接ファイルを読み取る
- カバレッジデータがある場合は coverage-reporter サブエージェントの結果を活用する

## 検出対象

### 1. テストファイルが存在しないソースファイル

ソースファイルに対応するテストファイル（`XxxTests.swift`）が存在するかを確認する。

### 2. テストされていない public / internal メソッド

ソースファイルの public / internal メソッドがテストコード内で呼び出されているかを確認する。

### 3. 未テストの分岐パス

- `if / else` 分岐の片方しかテストされていない
- `switch` 文の一部の `case` がテストされていない
- `guard` 文の早期リターンがテストされていない
- Optional のアンラップ（`nil` ケース）がテストされていない

### 4. エラーパスの未テスト

- `throws` 関数のエラーケースがテストされていない
- `Result` 型の `.failure` ケースがテストされていない
- `do-catch` のエラーハンドリングがテストされていない

### 5. 境界値の未テスト

- 数値の上限・下限
- 空配列・空文字列
- `nil` 値
- 日付の境界（月末、年末等）

## 検出手順

1. 対象のソースファイルを読み取る
2. テストディレクトリから対応するテストファイルを検索する
3. ソースファイルの public / internal メソッドを列挙する
4. テストファイル内でそのメソッドが呼び出されているかを Grep で確認する
5. ソースファイルの分岐構造（if / switch / guard）を解析する
6. テストコード内で各分岐パスがカバーされているかを判定する
7. 未カバーのパスをリストアップし、テストケースの追加を提案する

## 出力

```
## カバレッジギャップ検出: <対象ファイル名>

### テストファイル: <テストファイルパス> (存在 / 未作成)

### メソッド別カバレッジ
| メソッド | テスト有無 | 正常系 | 異常系 | 境界値 |
|---|---|---|---|---|
| fetchUser() | YES | COVERED | GAP | GAP |
| validate() | YES | COVERED | COVERED | GAP |
| deleteUser() | NO | - | - | - |

### 未カバーパス: N 件
- [GAP] <メソッド名>: <未カバーの説明>
  - 分岐: if <条件> の else パスが未テスト
  - 提案: <テストケースの概要>

### エラーパスの未テスト: N 件
- [GAP] <メソッド名>: throws の <エラー型> ケースが未テスト

### 推奨アクション
1. <優先度の高い順にテスト追加の提案>
```

## テスト対象の特定

- 引数でファイルパスが指定された場合はそのファイルを対象にする
- 指定がない場合は `git diff` の変更ファイルから `.swift` ファイル（テストファイルを除く）を対象にする
