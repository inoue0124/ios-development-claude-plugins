# PR 説明文自動生成

PR テンプレートに基づき、差分解析から変更内容・影響範囲・テスト計画を含む説明文を自動生成する。

## ツール使用方針

- `gh` CLI で PR 情報やテンプレートを取得することを優先する
- `git diff` / `git log` で変更内容を取得する
- MCP が利用できない場合も CLI で実行可能

## 生成フロー

### Step 1: テンプレートの取得

以下の順で PR テンプレートを検索する:

1. `.github/pull_request_template.md`
2. `.github/PULL_REQUEST_TEMPLATE.md`
3. `docs/pull_request_template.md`

テンプレートが見つからない場合はデフォルトテンプレートを使用する。

### Step 2: 変更差分の分析

```bash
# ベースブランチとの差分を取得
git diff main...HEAD --stat
git diff main...HEAD --name-only
git log main..HEAD --oneline
```

以下の情報を抽出する:

- 変更ファイル一覧と変更行数
- コミット履歴からの変更概要
- 影響を受けるモジュール / Feature

### Step 3: 説明文の生成

差分とコミット履歴を分析し、テンプレートの各セクションに内容を埋めて説明文を生成する。

## デフォルトテンプレート

テンプレートが存在しない場合に使用する形式:

```markdown
## 概要

<!-- 変更の目的と概要 -->

## 変更内容

<!-- 具体的な変更のリスト -->

-

## 影響範囲

<!-- 変更が影響するモジュール・画面 -->

-

## テスト計画

<!-- どのようにテストしたか / テスト予定 -->

- [ ]

## スクリーンショット

<!-- UI 変更がある場合 -->

## その他

<!-- レビュアーへの補足事項 -->
```

## 出力

生成された PR 説明文をマークダウン形式で出力する。ユーザーは内容を確認・編集してから PR に適用できる。

```
## PR 説明文（生成結果）

<生成された説明文>

---
上記の内容を確認し、必要に応じて編集してください。
```

## 検査対象の特定

- 引数でベースブランチが指定された場合はそのブランチとの差分を使用する
- 指定がない場合は `main` ブランチとの差分を対象にする
