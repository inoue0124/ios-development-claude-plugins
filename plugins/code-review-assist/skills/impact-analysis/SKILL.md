# 影響範囲分析

PR で変更されたファイル・クラス・メソッドに対し、影響を受ける依存先と呼び出し元を特定する。
リグレッションリスクの高い箇所を明示し、テスト対象の優先順位付けを支援する。

## ツール使用方針

- PR 差分の取得は `gh` CLI を使用する（`gh pr diff`）
- 呼び出し元の検索は Grep ツールを使用する
- ファイル読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read / Grep ツールで直接ファイルを読み取る

## 入力

- PR 番号またはブランチ名が指定された場合はそれを使用する
- 指定がない場合はカレントブランチの差分（`git diff main...HEAD`）を対象にする

## 分析手順

### Step 1: 変更の特定

1. 差分から変更されたファイル一覧を取得する
2. 各ファイルで変更された型（class / struct / enum / actor / protocol）を特定する
3. 変更されたメソッド・プロパティのシグネチャを抽出する

```bash
# 変更ファイル一覧の取得
gh pr diff <PR番号> --name-only

# カレントブランチの場合
git diff main...HEAD --name-only
```

### Step 2: 直接の影響先（呼び出し元）の追跡

変更された型・メソッドを参照している箇所をプロジェクト内から検索する。

- 型名での Grep による参照箇所の検出
- メソッド名での Grep による呼び出し箇所の検出
- Protocol 準拠の実装先の検出
- `import` 文からのモジュール依存の把握

### Step 3: 間接的な影響の推定

直接の呼び出し元がさらに上位層で使われているケースを 1 段階まで追跡する。

- View → ViewModel → UseCase の依存チェーンを辿る
- Protocol 変更の場合は全準拠型への影響を確認する

### Step 4: リグレッションリスクの評価

| リスクレベル | 条件 |
|---|---|
| High | Protocol のインターフェース変更（メソッド追加・シグネチャ変更） |
| High | 共有モデル（Entity / DTO）のプロパティ変更 |
| High | DI 登録の変更 |
| Medium | ViewModel のパブリック API 変更 |
| Medium | Repository の実装変更 |
| Low | View のレイアウト変更のみ |
| Low | 内部実装のリファクタリング（インターフェース不変） |

- <ファイル名> (<レイヤー>) - <参照箇所の説明>
