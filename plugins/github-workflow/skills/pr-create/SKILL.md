# PR 作成（差分解析 + 説明文生成）

> このスキルは `/pr-create` で明示的に実行します。GitHub に PR を作成する副作用があるため、自動活性化されません。

現在のブランチの差分を解析し、構造化された説明文付きの PR を作成する。

## ツール使用方針

- `gh` CLI を使用して PR を作成する
- `git` コマンドで差分・コミット履歴を取得する

## 実行フロー

### Step 1: ブランチ情報の収集

```bash
# 現在のブランチ名
git branch --show-current

# ベースブランチとの差分サマリー
git diff --stat main...HEAD

# コミット履歴
git log --oneline main...HEAD

# 変更ファイル一覧
git diff --name-only main...HEAD
```

### Step 2: 差分解析

差分の内容から以下を分析する。

- **変更の種別**: 新機能 / バグ修正 / リファクタリング / ドキュメント / テスト
- **影響範囲**: 変更されたモジュール・レイヤー（View / ViewModel / Model / Repository 等）
- **変更の規模**: 追加行数・削除行数・変更ファイル数

### Step 3: 説明文の生成

PR テンプレートがある場合はそれに従い、ない場合は以下の構造で生成する。

```markdown
## 概要
（変更の目的と概要を 1-3 文で）

## 変更内容
- 変更点 1
- 変更点 2

## 影響範囲
- 変更されたモジュール・レイヤー

## テスト
- [ ] ユニットテスト
- [ ] 動作確認

## 関連 Issue
closes #<issue-number>
```

### Step 4: PR テンプレートの確認

```bash
# PR テンプレートの存在確認
cat .github/pull_request_template.md 2>/dev/null || echo "テンプレートなし"
```

テンプレートが存在する場合はその形式に沿って説明文を生成する。

### Step 5: PR の作成

ユーザーに説明文を確認させた後、PR を作成する。

```bash
gh pr create \
  --title "<タイトル>" \
  --body "<本文>" \
  --base main
```

## 出力

```
## PR 作成完了

- URL: <PR URL>
- タイトル: <タイトル>
- ベースブランチ: <ベースブランチ>
- 変更ファイル数: N
- 追加行数: +N / 削除行数: -N
```

## 注意事項

- ベースブランチはデフォルトで `main` を使用する。ユーザーが指定した場合はそれに従う
- リモートに push されていない場合は push を促す
- コミットが 1 つの場合はコミットメッセージをタイトルに活用する
