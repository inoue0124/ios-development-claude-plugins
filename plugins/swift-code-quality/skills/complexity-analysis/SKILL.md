# 循環的複雑度・関数行数分析

指定されたファイルの関数・メソッドを解析し、循環的複雑度と行数を計測する。
複雑度が高い関数を特定し、リファクタリングの優先順位を提案する。
判定基準の詳細は → **references/COMPLEXITY_THRESHOLDS.md** を参照。

## ツール使用方針

- ファイル読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read / Grep ツールで直接ファイルを読み取る
- SwiftLint の `cyclomatic_complexity` / `function_body_length` ルールの出力も参考にする

```bash
# CLI フォールバック: SwiftLint で複雑度関連の違反を抽出
swiftlint lint --path <ファイルパス> --reporter json | python3 -c "
import json, sys
data = json.load(sys.stdin)
for v in data:
    if v['rule_id'] in ('cyclomatic_complexity', 'function_body_length', 'type_body_length', 'file_length'):
        print(f\"{v['severity']}: {v['file']}:{v['line']} - {v['rule_id']}: {v['reason']}\")
"
```

## 分析手順

### 1. 関数・メソッドの抽出

対象ファイルを読み取り、以下のパターンを検出する。

- `func <関数名>` — 通常の関数・メソッド
- `init(` — イニシャライザ
- `var <プロパティ名>: <型> {` — computed property
- `subscript(` — サブスクリプト

### 2. 循環的複雑度の計測

各関数内の以下の分岐構造をカウントする。

| 構造 | 複雑度加算 |
|---|---|
| `if` / `else if` / `guard` | +1 |
| `switch case` (各 case) | +1 |
| `for` / `while` / `repeat` | +1 |
| `&&` / `\|\|` (論理演算子) | +1 |
| `catch` | +1 |
| `?:` (三項演算子) | +1 |

基本複雑度は 1 から開始する。

### 3. 関数行数の計測

関数の開始行（`func` 行）から終了行（対応する `}`）までの行数を計測する。
空行・コメント行は行数に含める。

### 4. 判定

| メトリクス | 良好 | 注意 | 危険 |
|---|---|---|---|
| 循環的複雑度 | 1-10 | 11-20 | 21+ |
| 関数行数 | 1-40 | 41-100 | 101+ |
| 型の行数 | 1-200 | 201-350 | 351+ |
| ファイル行数 | 1-400 | 401-1000 | 1001+ |

## 出力

```
## 複雑度分析結果: <ファイル名>

### サマリー
- 関数数: N
- 複雑度 "注意" 以上: N 件
- 行数 "注意" 以上: N 件

### 関数別メトリクス（複雑度降順）
| 関数名 | 行数 | 循環的複雑度 | 判定 |
|---|---|---|---|
| <関数名> | N | N | 危険 / 注意 / 良好 |

### リファクタリング提案
- [危険] <関数名> (複雑度: N, 行数: N)
  - 提案: <分割方針の具体的な提案>
- [注意] <関数名> (複雑度: N, 行数: N)
  - 提案: <改善案>
```

## 検査対象の特定

- 引数でファイルパスが指定された場合はそのファイルを検査する
- 指定がない場合は `git diff` の変更ファイルから `.swift` ファイルを対象にする
