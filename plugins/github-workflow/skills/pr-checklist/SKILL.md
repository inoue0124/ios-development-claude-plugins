# PR セルフレビューチェックリスト

現在の PR または差分に基づいて、マージ前に確認すべきセルフレビューチェックリストを生成する。

## ツール使用方針

- `gh` CLI を使用して PR 情報を取得する
- `git` コマンドで差分を取得する

## 実行フロー

### Step 1: 差分情報の取得

```bash
# PR が存在する場合は PR の差分を取得
gh pr diff <pr-number>

# PR がない場合はブランチの差分を取得
git diff main...HEAD
```

### Step 2: 共通チェック項目

全ての PR に対して以下をチェックする。

#### コード品質

- [ ] 不要なデバッグコード（`print`, `debugPrint`, `#if DEBUG` 内の仮実装）が残っていないか
- [ ] TODO / FIXME / HACK コメントに対応済みか、または issue が作成されているか
- [ ] force unwrap (`!`) や force cast (`as!`) が不必要に使われていないか
- [ ] マジックナンバーが定数に抽出されているか

#### 命名・可読性

- [ ] 変数名・関数名が Swift API Design Guidelines に準拠しているか
- [ ] 新規追加したパブリック API にドキュメントコメントがあるか
- [ ] 不要なコメントアウトされたコードが残っていないか

#### アーキテクチャ

- [ ] MVVM のレイヤー分離が守られているか（View にビジネスロジックが含まれていないか）
- [ ] 新しい依存が適切に DI で注入されているか
- [ ] `@Observable` が使用されているか（`ObservableObject` / `@Published` でないか）

### Step 3: 差分に応じた動的チェック項目

変更内容に応じて追加のチェック項目を生成する。

| 変更の種類 | 追加チェック項目 |
|---|---|
| UI 変更 | アクセシビリティ対応、ダークモード対応、多言語対応 |
| API 通信 | エラーハンドリング、タイムアウト設定、リトライ処理 |
| データモデル変更 | マイグレーション対応、下位互換性 |
| 並行処理 | Sendable 準拠、actor 分離、タスクキャンセル対応 |
| テスト追加 | カバレッジ、エッジケース、モックの適切性 |
| 依存追加 | ライセンス確認、最小バージョン要件 |

## 出力

```
## セルフレビューチェックリスト

### 共通チェック
- [ ] デバッグコードの除去
- [ ] TODO / FIXME の確認
- [ ] force unwrap / force cast の確認
- [ ] 命名規則の準拠
- [ ] ドキュメントコメント

### アーキテクチャ
- [ ] MVVM レイヤー分離
- [ ] DI パターン
- [ ] @Observable 使用

### 差分固有チェック（検出された変更に応じて動的生成）
- [ ] <変更に応じたチェック項目>

### テスト
- [ ] 新規コードに対するテストが追加されているか
- [ ] 既存テストが変更後も通るか
```

## 対象の特定

- 引数で PR 番号が指定された場合はその PR を対象にする
- 指定がない場合は現在のブランチの差分を対象にする
