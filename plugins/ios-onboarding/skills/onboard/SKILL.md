# オンボーディングガイド一式生成

> このスキルは `/onboard` で明示的に実行します。自動活性化されません。

新メンバーがプロジェクトに参加した際に必要な情報を一括で生成するワークフロー。
個別スキル（codebase-overview, module-guide, architecture-map, glossary-gen, recent-changes-summary）と
codebase-analyzer サブエージェントを組み合わせて実行する。

## 実行フロー

### Step 1: コードベース分析（subagent）

codebase-analyzer サブエージェントを起動し、プロジェクト全体を走査する。
結果としてディレクトリ構造・モジュール構成・主要な型・依存関係・用語候補・コミット履歴を受け取る。

### Step 2: 個別ガイドの生成

サブエージェントの分析結果をもとに、以下のスキルを順次実行する。

1. **codebase-overview** -- プロジェクト構造の概要
2. **module-guide** -- 各モジュールの責務と依存関係
3. **architecture-map** -- アーキテクチャ構造のテキスト図
4. **glossary-gen** -- プロジェクト固有の用語集
5. **recent-changes-summary** -- 直近の変更要約

### Step 3: 統合ガイドの生成

全スキルの結果を統合し、新メンバー向けの包括的なオンボーディングガイドを生成する。

## 出力形式

```

## 目次

### セットアップ手順
1. リポジトリをクローンする
2. 依存関係を解決する（SPM の場合は `swift package resolve`）
3. Xcode でプロジェクトを開く
4. ビルドして動作確認する

### 最初に読むべきファイル
1. <ファイルパス> - <理由>
2. <ファイルパス> - <理由>
3. <ファイルパス> - <理由>

### 開発フロー

- PR ルール: <検出されたパターン>
- CI/CD: <検出されたパターン>

### 困ったときは
- <チームの連絡先やドキュメントリンク等、検出できた情報>
```

## 注意事項

- 生成されたガイドはあくまで自動分析に基づく推定であり、チームの実際の運用と異なる場合がある
- 生成後、チームメンバーにレビューしてもらうことを推奨する
