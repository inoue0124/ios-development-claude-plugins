# タスクリスト生成

詳細設計書（DESIGN.md）をもとに、実装タスクの順序付きリスト（TASKS.md）を生成する。
ドキュメントテンプレートの詳細は → **references/TASKS_TEMPLATE.md** を参照。

## ツール使用方針

- 既存コードの読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read / Grep / Glob ツールで直接ファイルを読み取る

## 入力

- **フィーチャー名**（例: `UserProfile`）
- **詳細設計書**（`docs/features/<feature-name>/DESIGN.md`）

## 前提条件

- 詳細設計書（DESIGN.md）が存在すること
- 存在しない場合は design-gen スキルの実行を先に案内する

## 実行手順

### 1. 詳細設計書の読み込み

`docs/features/<feature-name>/DESIGN.md` を Read ツールで読み取る。

### 2. タスク分解

詳細設計書の各セクションから実装タスクを導出する。以下の順序で分解する。

1. **Model 層** — データモデルの定義
2. **Repository 層** — Protocol 定義 → 具象実装
3. **UseCase 層** — ビジネスロジック
4. **ViewModel 層** — 状態管理・アクション
5. **View 層** — UI コンポーネント → 画面
6. **ナビゲーション** — ルーティング・画面遷移
7. **DI** — 依存注入の設定
8. **テスト** — ユニットテスト・UI テスト

### 3. タスクリストの生成

各タスクに以下の情報を付与する。

- タスク番号
- タスク名
- 対象ファイルパス
- 実装内容の概要
- 依存タスク（ある場合）

### 4. 出力

生成した TASKS.md の内容をユーザーに提示し、確認を得てからファイルに書き込む。
配置先はユーザーの iOS プロジェクト内とする。

```
docs/features/<feature-name>/TASKS.md
```

## タスク分解の原則

- **依存関係順** — 下位レイヤー（Model → Repository）から上位レイヤー（View）へ
- **1 タスク 1 ファイル** — 原則として 1 タスクで 1 ファイルを作成・変更する
- **テスタブル単位** — 各タスク完了時点でコンパイルが通る粒度にする
- **チェックボックス** — `- [ ]` で未完了、`- [x]` で完了を管理する

## 出力形式

```
## タスクリスト生成結果

### 出力ファイル
- docs/features/<feature-name>/TASKS.md

### タスクサマリー
- 総タスク数: X 件
- Model 層: X 件
- Repository 層: X 件
- UseCase 層: X 件
- ViewModel 層: X 件
- View 層: X 件
- ナビゲーション: X 件
- DI: X 件
- テスト: X 件

### 次のアクション
- [ ] タスクリストの内容を確認・承認する
- [ ] タスク順に実装を開始する（/implement-feature を推奨）
```
