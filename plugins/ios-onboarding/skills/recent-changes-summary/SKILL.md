# 最近の変更要約

直近のコミット履歴・PR・ブランチの状況を分析し、「今プロジェクトで何が起きているか」を要約する。新メンバーが現在の開発状況をキャッチアップするために使用する。

## ツール使用方針

- Git 操作は CLI（`git` コマンド）を使用する
- GitHub の PR・Issue 情報は `gh` CLI を優先する
- ファイル読み取りが必要な場合は xcodeproj-mcp-server を優先し、フォールバックで Read / Grep を使用する

## 分析手順

1. `git log --oneline --since="2 weeks ago"` で直近 2 週間のコミット一覧を取得する
2. `git log --format="%an" --since="2 weeks ago" | sort | uniq -c | sort -rn` でアクティブなコントリビューターを把握する
3. `git log --format="" --name-only --since="2 weeks ago" | sort | uniq -c | sort -rn | head -20` で変更頻度の高いファイルを特定する
4. `git branch -r --sort=-committerdate | head -10` でアクティブなブランチを確認する
5. `gh pr list --state open --limit 10` でオープン中の PR を確認する（`gh` が利用可能な場合）
6. `gh issue list --state open --limit 10` でオープン中の Issue を確認する（`gh` が利用可能な場合）
7. コミットメッセージのパターンからトピック（機能追加・バグ修正・リファクタリング等）を分類する

## 出力形式

```
## 最近の変更要約（過去 2 週間）

### サマリー
<1-2 文でプロジェクトの現在の状況を要約>

### 開発中の主な機能・トピック
1. **<トピック名>** - <概要>
   - 関連コミット: N 件
   - 関連ファイル: <主要ファイル>
2. **<トピック名>** - <概要>
   ...

### 変更統計
- 総コミット数: N
- アクティブなコントリビューター: N 名
- 変更ファイル数: N

### 変更頻度の高いファイル（ホットスポット）
| ファイル | 変更回数 | 推定される作業内容 |
|---|---|---|
| <ファイルパス> | N | <コミットメッセージから推定> |

### オープン中の PR
| # | タイトル | 作成者 | 状態 |
|---|---|---|---|
| <番号> | <タイトル> | <作成者> | <レビュー待ち / 変更要求 等> |

### オープン中の Issue
| # | タイトル | ラベル | 優先度 |
|---|---|---|---|
| <番号> | <タイトル> | <ラベル> | <推定> |

### アクティブなブランチ
| ブランチ名 | 最終更新 | 推定される作業内容 |
|---|---|---|
| <ブランチ名> | <日時> | <ブランチ名から推定> |

### 新メンバーへの注意事項
- <現在進行中の大きな変更やリファクタリングがあれば注記>
- <マージ待ちの破壊的変更があれば注記>
```

## 対象の特定

- 引数で期間が指定された場合はその期間を使用する（例: `--since="1 month ago"`）
- 指定がない場合は直近 2 週間を対象にする
