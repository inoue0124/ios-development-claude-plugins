# 未使用コード・未使用 import 検出

指定されたファイルまたはモジュール内の未使用コード（デッドコード）と未使用 import を検出する。
コードベースの不要な肥大化を防ぎ、メンテナンス性を向上させる。

## ツール使用方針

- ファイル読み取りは xcodeproj-mcp-server の利用を優先する
- MCP が利用できない場合は Read / Grep ツールで直接ファイルを読み取る

## 検出項目

### 1. 未使用 import

ファイル内で import されているモジュールのうち、実際に使用されていないものを検出する。

```swift
// 検出例: UIKit を import しているが使用していない
import Foundation
import UIKit       // [WARN] 未使用の import
import SwiftUI

struct ContentView: View { ... }  // SwiftUI のみ使用
```

**検出手法:**

1. ファイル内の全 `import` 文を抽出する
2. ファイル内のコードから各モジュールの型・関数・プロトコルの使用を検索する
3. 使用が見つからない import を未使用として報告する

**注意:** 以下の import は誤検出しやすいため慎重に判定する。

- `@testable import` — テストファイルでの使用
- `import Foundation` — 暗黙的に使われる場合がある
- 拡張（extension）で使用されるモジュール

### 2. 未使用の関数・メソッド

`internal` 以下のアクセスレベルで、プロジェクト内から呼び出されていない関数を検出する。

```swift
// 検出例: 他のどこからも呼ばれていない関数
func unusedHelper() -> String {  // [WARN] 未使用の関数
    return "unused"
}
```

**注意:** 以下は誤検出しやすいため除外する。

- `public` / `open` な API（外部モジュールから使用される可能性がある）
- `@objc` / `@IBAction` / `@IBOutlet` 属性付き（ランタイムから呼び出される）
- Protocol 準拠のメソッド
- `override` メソッド
- `CodingKeys` 等の特殊な型

### 3. 未使用のプロパティ

宣言されているが読み取り・書き込みされていないプロパティを検出する。

### 4. 未使用の型定義

宣言されているがインスタンス化・参照されていない型を検出する。

### 5. 未使用のプロトコル

宣言されているが準拠する型が存在しないプロトコルを検出する。

## 出力

```
## 未使用コード検出結果: <対象パス>

### サマリー
- 未使用 import: N 件
- 未使用関数: N 件
- 未使用プロパティ: N 件
- 未使用型: N 件
- 未使用プロトコル: N 件

### 未使用 import
- [WARN] <ファイル>:<行番号> - `import <モジュール名>` は使用されていません
  [提案] この import 文を削除してください

### 未使用コード
- [WARN] <ファイル>:<行番号> - `<関数名/型名>` は参照されていません
  [提案] 不要であれば削除してください。将来使用予定がある場合はコメントで意図を記載してください

### 注意
- `public` / `open` な API はプロジェクト外部から使用される可能性があるため検出対象外です
- 誤検出の可能性がある項目は [INFO] として表示します
```

## 検査対象の特定

- 引数でファイルパスが指定された場合はそのファイルを検査する
- 指定がない場合は `git diff` の変更ファイルから `.swift` ファイルを対象にする
